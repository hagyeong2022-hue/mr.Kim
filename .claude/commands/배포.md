# 배포 명령어

프로젝트의 변경사항을 자동으로 커밋하고 GitHub에 배포합니다.

## 실행 절차

### 1️⃣ 변경된 파일 확인

변경된 파일 목록을 확인합니다.

```bash
git status --short
```

### 2️⃣ 커밋 메시지 자동 생성

변경된 파일들을 분석하여 자동으로 커밋 메시지를 생성합니다.

```bash
git diff --cached --stat
```

### 3️⃣ 커밋 및 푸시

모든 변경사항을 스테이징하고 커밋한 후 GitHub에 푸시합니다.

```bash
git add -A
git commit -m "$(date +'%Y-%m-%d %H:%M:%S') - 배포"
git push origin main
```

### 4️⃣ 배포 완료 확인

배포 완료 상태를 확인합니다.

```bash
git log --oneline -1 && echo "✅ 배포 완료!"
```

## 사용 방법

터미널에서 다음 명령어를 실행하세요:

```
/배포
```

## 주의사항

- 커밋 전에 변경사항이 올바른지 확인하세요
- GitHub 인증이 필요합니다
- 푸시 실패 시 네트워크 연결을 확인하세요
